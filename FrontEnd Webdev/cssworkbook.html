Units in css

Ref: Css unit testing

Notes:

  • Two Types of Units we have: Absolute (px) and Relative(%, em, rem)
  • Absolute units like px is fixed. It is now way depended upon anything.
  • Relative units are depended upon there parent font-size or container's width and height.

    • %: take's that much percentage of the parent container eg: width:50%; height:50%
    • em: this is relative to the font-size. if section has font-size 50px and the child element has font-size: 2em than it means the child font size will be 2*50px.
      For other things like margin and all 1em will be equal to the font-size of that element.
    • One Drawback of ems: They always based on the parent font-size so if have nested ul or element than as you dive deep in the list smaller or bigger the font-size will be.(avoid it !)
    • rem: (Root em) This will solve the issue with ems, the font-size of the child element will always be based on the root parent element.

Other Most used property in css

eg: opacity, transitions, background properties

Ref: Other Css Properties

Notes:

  • rgba --> rgb is the normal color formate each varies from 0 to 255, a (alfa channel) controls the opacity(0-> transparent and 1-> opace). Only the background is affected.
  • opacity --> It's Value also ranges from 0(transparent) to 1(Opace) but the entire content is affected.
  • position --> used to set the position of an element. eg: static, relative, absolute, fixed and sticky.
    • static--> Its position in the document is fixed i.e (top, bottom, left and right will not work)
    • relative --> ITs position is relative to other element i.e (top, bottom, left and right will work)
    • Absolute(Imp) --> The Enitre space collapse after this element and top, bottom, left and right will position wrt to the body. but If the parent has position other than static than it will position based on the parent.
    • Fixed --> same as absolute just it will never be relative any parent or so. It's position is always fixed wrt entire page.(Kind of if you scroll also it's position will be fixed.)
      eg: Used for fixing nav bars fixed on top while scrolling
    • sticky: It same as fixed but the position will fixed only when you start scrolling.
  • Transition: We can animate a css property. Four fields it takes: propertyname, duration, timing-function and delay.
    Important here is the timing function that defined how it should trnasit from one state to another eg: ease-in, ease-out, linear etc.(Commonly used for hover)
  • Transform: eg (all this have x,yz axis like rotateX, scaleX)
    • rotate(45deg) using transfrom origin we can control the point of rotation.
    • Scale(0.5) using this we can scale the element, scale(2,1) width will be twice and height will be same.
    • translate it is used to move the element around, tanslate(x,y) move x unit horizontally and y unit vertically
    • Skew(10deg) IT wil skew the element, also skew(x,y) available.

    Note: All this can be used atonce in one line like: transform: rotate(200deg) scale(1.3) translateX(300px)

Note: Whenever any property is written in one line eg: boder: 5px solid #000; the same thing can be breakdown and can be written like
border-width: 5px
border-style: solid
border-color: #000 (black)
(Order Really matters here) Same thing can be donw with other properties that allows multiple values in the same line. eg: padding, margin, transition, transform etc

Flex Box in Css

Click here to see the flexBox Click here

    Notes:
  • flexbox are just used to arrange the elemnets inside the space.
    eg:display : flex (Always define in some box element like section, div, mian etc)
  • flexbox has main-axis(horizontal) and cross-axis(vertical) --> Flex direction decides the dir of the flex-properties inside a flex.
  • flx-dir: row(left to right -> main-axis & top-bottom--> cross-axis), column(left to right -> cross-axis & top-bottom--> main-axis)
    • Note: All this are based on the dir of the flx direction.
  • justify content will have options to arrange the contents in the main-axis. like evenly space btn them or start from begin or end etc
  • flex Wrap--> If the content is too much and its sqizing than use wrap to keep everthing within the flex. use wrap or wrap-reverse. flex wrap used to reverse the dir of cross-axis box
  • Align-items with will arrange the items in the cross-axis
  • Align-content will handle the spacing between the element on cross-axis.eg: flex-end, flex-start, space-between (This will only work if we have flx wrap )
  • align self is used to align a single element in a cross axis.
    • (Below properties are defined for the elements inside the flexbox)
  • flex basic will define the intial size of the element that is added in a flex-box
  • flex grow will allow an element to take all of the additional space in the flex container.
  • flex sink will allow the elemnt to srink if enough space is not available in the flex contanier. (Higher the number more faster the element will shrink)
  • flex This property can define all 3 of the top grow, shrink and basis > in one go (Follow this order))
